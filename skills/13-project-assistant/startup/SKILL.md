# startup — 開工

當使用者說「**開工**」時，請執行以下流程。

## 步驟 0：檢查是否為新專案

檢查目前目錄是否存在專案設定檔（`PROJECT.md` 或 `AGENTS.md`）：

- **如果存在** → 讀取設定，繼續步驟 1
- **如果不存在** → 詢問使用者：
  > 「這個資料夾沒有專案設定檔，你是要：
  > 1️⃣  啟動新專案 — 幫你建立結構 + 設定檔
  > 2️⃣  延續舊專案 — 掃描現有檔案，自動產生設定檔」
  - 選 1 → 執行 project-init 流程（5 種專案模板可選）
  - 選 2 → 掃描目錄中所有檔案，自動產生 PROJECT.md 草案供確認

## 步驟一：檢查專案設定檔

如果設定檔存在，讀取並了解專案在做什麼、目前進度到哪裡。

## 步驟二：如果設定檔存在

1. **讀取專案設定檔**，了解這個專案在做什麼、目前進度到哪裡
2. **檢查 Obsidian 知識庫**中是否有對應的待辦事項
3. **回報給使用者**。用輕鬆的口吻，例如：
   > 「早安！目前專案「個人網站」的進度是首頁已完成、關於我頁面還沒做。今天的待辦有：完成關於我頁面、加上聯絡表單。要開始嗎？」

## 步驟三：如果設定檔不存在（已於步驟 0 處理）

此步驟為備援流程，若使用者選擇跳過步驟 0 的引導則執行：

1. **檢查目錄是否為空**
2. **若為空**：詢問使用者是否要初始化新專案
3. **若非空**：掃描目錄中有哪些檔案，自動產生一份專案設定檔草案

## 新手友善提醒

- 用白話中文解釋你在做什麼
- 不要一次問太多問題
- 如果使用者看起來困惑，主動問「需要我解釋一下嗎？」
